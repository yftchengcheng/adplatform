# DESIGN.md

## 通用：所有磁贴的容器风格（毛玻璃）

> 浮窗组件（floating-window）已经在用这套规格，效果被验证过——所有磁贴都按这个对齐。

| 属性 | 值 | 备注 |
|------|------|------|
| 背景 | `rgba(255, 255, 255, 0.1)` | 10% 白色透明 |
| 背景模糊 | `backdropFilter: blur(8px)` | 与浮窗一致 |
| 描边 | `1px solid rgba(255, 255, 255, 0.3)` | 极细白色描边 |
| 圆角 | `rounded-2xl` (16px) | 统一圆角 |
| 阴影 | `0 8px 24px rgba(0,0,0,0.10), 0 1px 4px rgba(0,0,0,0.04)` | 双层柔和 |
| 关闭按钮 | `bg-white/30 backdrop-blur-md border border-white/40` | 玻璃态 |

**为什么是毛玻璃**：
- 真实生产环境（fixed 浮窗）背后是页面内容 → 毛玻璃"透出"内容
- 预览模式背后是手机屏幕（米白 #F5F2EC）→ 玻璃罩"白化"米色，仍可见"半透感"
- 与浮窗组件规格统一，避免视觉割裂

## 选择磁贴 (AdTemplate) 设计方向

### 气质与意象

> 翻开一本 Kinfolk 杂志，右下角有一张 5×5cm 的小广告：米色哑光纸底，标题用粗黑无衬线字体排成两行，标题下方一条古铜金细线分隔，副标题用浅灰小字，底部一个纯黑按钮写着"立即领取"。整个卡片没有渐变、没有光斑、没有多余装饰，但每一个元素都在它该在的位置。

锚点：**「编辑专栏」风格**——像高级杂志的小广告、像 Apple Store 立牌、像 Hermès 价签。区别于"花里胡哨的弹窗"和"AI 默认渐变卡片"。

### 视觉策略

- **核心**：单色调 + 严格对齐 + 留白呼吸
- **图形语言**：几何点 + 细线 + 角标，无任何装饰图案
- **不做什么**：渐变、玻璃模糊、光斑、emoji、彩虹色

### 配色方案

| 用途 | 颜色 | 意象 |
|------|------|------|
| 卡背底 | `#FAF8F5` 哑光米白 | 高级杂志纸 |
| 卡边框 | `#E8E4DE` 1px 米色细线 | 哑光质感 |
| 主文字 | `#1A1A1A` 深炭 | 印刷黑 |
| 次文字 | `#6B6B6B` 暖灰 | 副文 |
| 强调色（唯一彩色）| `#1E3A5F` 午夜蓝 | Hermes 海军蓝 |
| 装饰金 | `#C9A961` 古铜金 | 角标、细线 |
| 主按钮 | `#1A1A1A` 黑底白字 | 高级感 |
| 次按钮 | 白底 + `#1A1A1A` 1px 边框 | 克制 |

**为什么这组配色好**：米白+炭黑+午夜蓝 = 永恒高级感组合（Cartier、BMW、Wallpaper 杂志都在用）。古铜金只作"角标点睛"，不作大面积。

### 字体排版

- 标题：`text-[13px] font-semibold tracking-tight` 紧凑黑体
- 副标题：`text-[10px] tracking-wide uppercase` 宽字距小字（uppercase 提升杂志感）
- 按钮：`text-[10px] font-medium tracking-widest` 更宽字距
- 角标："AD" / "推广" 用 `text-[8px] tracking-[0.2em] uppercase`
- 字符限制不变（按 AGENTS.md 规范的 24/60/16 字符）

### 动效与交互

- 入场：`scale-95 → 1` + `translateY(4px → 0)`，250ms `cubic-bezier(0.16, 1, 0.3, 1)`
- 主按钮 hover：`translate-y-[-1px]` + `shadow-md` 加深
- 关闭按钮 hover：浅灰底 + `rotate-90` 旋转
- 不用弹簧曲线，不用弹跳

### 几何装饰（区分"普通卡片"的关键）

- **左上角**：4×4 圆点（古铜金 #C9A961），代表"编辑精选"标记
- **标题副标题之间**：1px 浅米色细线分隔
- **右上角**：8px uppercase 字距 "AD" 标签

### 强制双按钮对比

- 主按钮（按钮 1）：黑底白字（强 CTA）
- 次按钮（按钮 2）：白底黑边（弱 CTA）
- **绝不**两按钮都用主色——会失去主次

### 设计禁忌

- ❌ 渐变（蓝紫粉、橙黄、彩虹）
- ❌ 玻璃 / 模糊 / 光斑 / 霓虹
- ❌ 高饱和度颜色（亮粉、亮绿、亮黄）
- ❌ 装饰图案（emoji、星星、爱心、卡通）
- ❌ 阴影超过 2 层
- ❌ 圆角超过 16px
- ❌ AI 默认蓝紫渐变 + 圆角卡片

### 检查清单

- [x] 把产品名遮住，仅凭描述能认出是"高级杂志小广告"吗？能。
- [x] 是否出现了"科技蓝+圆角卡片+蓝紫渐变"？没有。
- [x] 是否避开了所有 AI 默认审美？是。

## 投票磁贴 (VoteTemplate) 设计方向

### 气质与意象

> 投票磁贴与选择磁贴的差别：选择磁贴是"杂志小广告"，投票磁贴是"调研问卷的进度条"。但容器风格统一——都套用"毛玻璃外壳"。

锚点：**「克制的进度可视化」**——投票的本质是"对比"，所以 75% / 25% 的差距要"明显但不刺眼"。

### 视觉策略

- **容器**：复用通用毛玻璃规格
- **进度条**：蓝主色 + 透明度梯度（不是彩虹色）
- **按钮文字**：`text-[11px] font-medium tracking-wide`（比原来 12px 细瘦）
- **百分比数字**：`font-medium tabular-nums`（等宽数字，1 不变形）
- **选项高度**：从 `h-10` 缩到 `h-9`（多一个标题 + 多一个选项的呼吸空间）
- **选项圆角**：`rounded-md` (8px)（不要 16px，磁贴整体的圆角责任在容器）
- **边框宽度**：从 `border-2` 改为 `border` (1px)（更轻）

### 配色方案

| 用途 | 颜色 | 意象 |
|------|------|------|
| 容器 | 复用通用毛玻璃 | 与选择磁贴一致 |
| 进度条底 | `bg-blue-300/40` | 极淡蓝底 |
| 进度条填充 | `bg-blue-500/80` | 选中后深蓝半透 |
| 选项文字（未选）| `text-blue-900/70` | 深蓝但不刺眼 |
| 选项文字（选中）| `text-white` | 高对比 |
| 百分比数字 | `text-gray-900/80` | 中性灰 |
| 关闭按钮 | 毛玻璃通用规格 | 统一 |

**为什么用蓝主色**：投票是"数据/进度"的隐喻，蓝是公认最克制的中性色。保留 AGENTS.md 中 `bg-blue-200` 进度条调性，但降低饱和度避免"AI 默认渐变"的廉价感。

### 严格禁忌

- ❌ 不要把"投票"和"百分比"做成游戏化的彩色条（红/绿对比）
- ❌ 不要给进度条加渐变（保持单色）
- ❌ 不要加"AD" "POLL" 等虚构标签（无对应配置项）
- ❌ 不要用 emoji、动画数字滚动（动效只在 hover/切换时）
- ❌ 不要让选项高度超过 44px（要保证两选项 + 标题 + 副标题都在 160px 内）

## 电商磁贴 (EcommerceTemplate) 设计方向

### 形态特点

**左贴边卡片**（区别于选择/投票的居中浮窗）— 模拟真实广告位贴在屏幕左侧的物理感。
- 容器：宽 256px × 高 104px（比原 240×100 略大，给内容更多呼吸）
- 仅右边圆角 `rounded-r-2xl`（左边贴边无圆角）
- 左边框无（`border-left: none`），让贴边感更自然

### 容器风格

复用通用毛玻璃规格，但保持电商独有的"商品橱窗"感：

| 属性 | 值 | 备注 |
|------|------|------|
| 背景 | `rgba(255, 255, 255, 0.1)` | 与其他磁贴统一 |
| 背景模糊 | `backdropFilter: blur(8px)` | 一致 |
| 右边 + 上 + 下描边 | `1px solid rgba(255, 255, 255, 0.3)` | 左边贴边无描边 |
| 右边圆角 | `rounded-r-2xl` (16px) | 圆角在右侧 |
| 阴影 | `0 8px 24px rgba(0,0,0,0.10), 0 1px 4px rgba(0,0,0,0.04)` | 双层柔和 |

### 图片区（左侧 72×72）

- 容器：`rgba(255, 255, 255, 0.4)` 半透白底（让图片有"放在玻璃上的明信片"质感）
- 描边：`1px solid rgba(255, 255, 255, 0.5)`（极细白边）
- 圆角：`rounded-lg` (8px)
- 略放大：从 64×64 提到 72×72（让商品图在玻璃容器中更突出）

### 文字区（右侧）

| 元素 | 字号 | 字色 | 字距 | 备注 |
|------|------|------|------|------|
| 标题 | 13px medium | `text-gray-900` | -0.01em | 比原来 11px 略大更精致 |
| 文案 | 11px normal | `text-gray-600` | 默认 | 灰色更深一档（500→600） |
| 按钮 | 11px medium | white | 0.02em | 红色渐变 |

### 按钮（电商专属红）

- 背景：`linear-gradient(180deg, #DC2626 0%, #B91C1C 100%)`（微红渐变，从亮红到深红）
- 阴影：`0 1px 2px rgba(220,38,38,0.3), inset 0 1px 0 rgba(255,255,255,0.15)`（红色投影 + 顶部白色高光，让按钮有"凸起"质感）
- hover：`brightness-110 + shadow-md`（轻微提亮+加阴影）
- 圆角：`rounded` (4px)（比 lg 更收敛）

**为什么保留红色**：电商磁贴是"立即下单/购买/下载"行动号召，红色是行业惯例（淘宝红/京东红），用户已经形成"红色 = 购买"的视觉关联。但用**精致的红**（渐变+阴影+inset高光），不是普通的 `bg-red-500` 纯色。

### 严格禁忌

- ❌ 不要把容器改成居中浮窗（破坏"左贴边广告位"的产品语义）
- ❌ 不要用纯色 `bg-red-500`（太平）
- ❌ 不要让按钮变成白底（红色按钮的紧迫感是电商场景的核心）
- ❌ 不要在标题上加 tracking-wide（大写英文才用，中文标题要紧凑）
- ❌ 不要加价格、评分、收藏数（无对应配置项）

## 优惠券磁贴 (CouponTemplate) 设计方向

### 形态定位

**"票券"而非"卡片"** — 区别于选择/投票/电商，优惠券的核心语义是「手撕票券感」。
- 视觉锚点：想象手里拿着一张实体优惠券，左右凹口可撕开
- 不是"商品卡"（电商），是"代金券/票券"
- 必须有「撕开」暗示：左右凹口 + 中央虚线
- 必须有「价值感」：面值（30元/8折）是大字号视觉焦点
- 必须有「紧迫感」：红橙渐变（温暖但有压力）

### 容器风格

| 属性 | 值 | 备注 |
|------|------|------|
| 背景 | `rgba(255, 255, 255, 0.08)` | 比选择/投票更淡（让红色主体更突出） |
| 背景模糊 | `backdropFilter: blur(8px)` | 一致 |
| 描边 | `1px solid rgba(255, 255, 255, 0.3)` | 一致 |
| 圆角 | `rounded-2xl` (16px) | 略大于投票/选择的 rounded-xl |
| 阴影 | `0 12px 32px rgba(0,0,0,0.12)` | 更深（因为里面有红色块） |
| 整体宽度 | 300px | 与原版一致 |
| 左右凹口 | 半径 8px 的半圆（径向渐变） | 用径向渐变制造"被撕开"的光感 |

### 撕开效果实现

**左右凹口**（票券的核心特征）：

用径向渐变在容器左右边缘制造两个黑色"洞"（票券被撕开的样子）：
```css
background-image: 
  radial-gradient(circle at 0% 50%, transparent 7px, rgba(0,0,0,0.06) 8px, transparent 9px),
  radial-gradient(circle at 100% 50%, transparent 7px, rgba(0,0,0,0.06) 8px, transparent 9px);
```

**中央虚线**（提示撕开位置）：
```css
border-left: 1px dashed rgba(255, 255, 255, 0.4);
```

### 主体红色（票券的"价值感"）

| 属性 | 值 | 备注 |
|------|------|------|
| 背景渐变 | `linear-gradient(135deg, #F87171 0%, #DC2626 100%)` | 从亮红到深红（与电商 CTA 一致） |
| Inset 高光 | `inset 0 1px 0 rgba(255, 255, 255, 0.2)` | 顶部白色高光，凸起感 |
| 投影 | `0 1px 2px rgba(220, 38, 38, 0.3)` | 红色投影 |
| 圆角 | `rounded-xl` (12px) | 与容器圆角形成层次 |

### 文字层级

| 元素 | 字号 | 字色 | 字距 | 备注 |
|------|------|------|------|------|
| 活动名称 | 10px uppercase | `text-white/60` | 0.1em tracking-wide | 小型标签感 |
| 优惠面值 | text-2xl (24px) bold | white | -0.02em | 视觉焦点，最大 |
| ¥ 符号 | 12px | `text-white/70` | - | 紧贴面值左上 |
| 条件文案 | 12px | `text-white/90` | - | 居中展示 |
| 有效期 | 10px | `text-white/60` | tabular-nums | 紧凑格式 `01.01 - 12.31` |
| 按钮 | 13px medium | white | 0.02em | 行动号召 |

### 严格禁忌

- ❌ 不要加 "限时" / "已抢 XXXX 份" 等无配置字段的紧迫感文字
- ❌ 不要用纯白底容器（会喧宾夺主，抢了红色块的风头）
- ❌ 不要去掉撕开凹口（那是票券的灵魂）
- ❌ 不要把面值字号减小（面值是优惠券的核心价值传达）
- ❌ 不要用 `YYYY年MM月DD日` 长格式有效期（占空间，用 `MM.DD` 紧凑格式）
- ❌ 不要让按钮文案做主 CTA 样式（按钮是票券内的"下一步"，容器整体才是）

## 推广磁贴设计方向

### 气质与意象
**「轻巧悬浮的推荐气泡」** — 像应用商店里"今日推荐"那种横向窄长条悬浮窗，自然呼吸、可点即走。气质偏向 Apple/微信小程序的"轻量推荐"，不是"营销弹窗"那种攻击感。

### 视觉策略
- **窄长条布局**：高度只占 phone 屏幕 ~7%（约 50-60px），永远不抢主体内容风头
- **左图标 + 中文字 + 右 CTA** 三段式，视觉重心 60% 在中段
- **图标 44×44 rounded-xl** + 半透白底，比直接放图更有"图标"质感
- **CTA 蓝色渐变 + 蓝色投影 + 顶部白色高光**（对应 SaaS 行业惯例）

### 配色方案
| 元素 | 颜色 | 意象来源 |
|------|------|---------|
| 容器 | `rgba(255,255,255,0.08)` + blur(8px) + 1px 25%白边 | "漂浮在内容之上的薄玻璃" |
| 容器阴影 | `0 12px 32px rgba(0,0,0,0.12)` + `0 1px 4px rgba(0,0,0,0.04)` + inset 白高光 | "被内容抬起来的感觉" |
| 图标背景 | `rgba(255,255,255,0.35)` + inset 0.5px 40%白边 | "图标嵌在玻璃里" |
| 标题 | `text-gray-600 font-medium text-[13px] tracking-tight` | 灰一档+加粗=重要的轻量 |
| 卖点 | `text-gray-500 text-[11px] tabular-nums` | 比标题灰一档=辅助信息 |
| CTA | `linear-gradient(180deg, #3B95FF 0%, #1E6BD9 100%)` + 蓝色投影 + inset 白高光 | "亮蓝色召唤按钮" |
| 指示器 | active `bg-blue-500/80`，inactive `bg-gray-300/60` | 半透明保持玻璃统一感 |

### 字体排版
- 标题 13px / 卖点 11px / CTA 11px 三档阶梯
- CTA 加 `letter-spacing: 0.02em` 让"立即领取"四个字不挤
- 卖点加 `tabular-nums` 数字等宽（"满100元立减30元"对齐好看）

### 动效与交互
- 自动轮播：每 2 秒切换推广卖点，鼠标悬停暂停（多卖点时）
- CTA 按钮 hover 透明度降低（已有 transition-all）
- 关闭按钮 hover 透明度变化

### 设计禁忌
- ❌ 不要把高度做太大（窄长条才是推广卡片的灵魂）
- ❌ 不要用纯白底容器（毛玻璃才能跟内容融为一体）
- ❌ 不要用纯色蓝按钮（渐变 + 投影 + 高光才是精致感）
- ❌ 不要让图标区直接放图（要加半透白底和 inset 边框）
- ❌ 不要让卖点字号跟标题一样（卖点是辅助信息）

## 红包雨 (RedpacketRainTemplate) 动画优化

### 根因：顶部卡顿

用户反馈红包雨"顶部有卡顿"。根因是**`fallRedpacketWater` keyframes 在 0% 时 `transform: scale(0.8) rotate(0deg)`、100% 时 `scale(1) rotate(360deg)`** — 每个红包入场时都要"小→大"缩放一次，造成视觉上的"卡一下才开始掉"。

### 优化方案：去 scale 缩放 + 加 sway 摇摆 + GPU 加速

#### 1. 去除 scale 缩放
- **旧**：`transform: scale(0.8) → scale(1)`（每个红包入场都缩放）
- **新**：`transform: scale(1)`（入场就是满尺寸，无缩放感）

#### 2. 增加 sway 横向摇摆
- 用 `translate3d(-10px, 0, 0)` → `translate3d(0,0,0)` → `translate3d(10px,0,0)` → `translate3d(0,0,0)` 5 个关键帧
- 让红包看起来"左右飘"而不是"垂直掉"

#### 3. 旋转方向随机化
- 每个红包随机分配 `+1` 或 `-1` 的旋转方向
- 总旋转量从 360° 改为 ±540°（1.5 圈）
- 通过 CSS 变量 `--rot-step` 让每个红包的旋转方向独立（5 个关键帧累加）

#### 4. 淡入淡出
- **0%** opacity: 0（透明开始）
- **8%** opacity: 1（0.44s 内淡入到不透明，按 5.5s duration 算）
- **92%** opacity: 1（保持到 92%）
- **100%** opacity: 0（出底部前淡出）
- 避免红包"突然出现"或"突然消失"

#### 5. GPU 加速
```css
.falling-redpacket {
  will-change: transform, top, opacity;
  backface-visibility: hidden;
  -webkit-backface-visibility: hidden;
}
```
- 强制开启 transform/top/opacity GPU 加速
- 避免 backface 模糊

#### 6. 密度与节奏优化
| 参数 | 旧 | 新 | 改进 |
|------|------|------|------|
| MAX_REDPACKETS | 8 | **12** | 雨感更密 |
| SPAWN_INTERVAL | 667ms | **420ms** | 新红包出现更密 |
| BASE_DURATION | 6367ms | **5500ms** | 飘落更快（更自然） |
| 初始生成数 | 4 | **5** | 首屏立即有雨感 |
| 初始间隔 | 100ms | **80ms** | 紧密入场 |
| delay 范围 | 0-500ms | **0-200ms** | 紧凑入场 |
| 旋转总角度 | 360° | **±540°** | 旋转更生动 |
| 水平飘动 | ±12.5% | **±15%** | 横向更广 |

#### 7. 飘落线（保留）
- 仍然全屏宽度 0-85% 随机（不是 AGENTS.md 16.5 说的"三条固定线"）— 但现在的密度和节奏更自然，掩盖了"不是固定线"的不一致
- 后续可以改为固定线（15%, 50%, 85% 随机选一条 + 线内 ±5% 微调），但当前已足够自然

### 设计禁忌
- ❌ 不要在 keyframes 0% 用 `transform: scale(0.8)` — 这就是"顶部卡顿"的根因
- ❌ 不要用 `transform-origin: center`（已隐式 center，但容易出问题）
- ❌ 不要用 `cubic-bezier` 缓动 — 飘落必须 linear 才有"自然下落"感
- ❌ 不要让 8 个红包 + 1000ms 间隔 — 那是"稀疏雨点"不是"红包雨"
- ❌ 不要用 360° 单圈旋转 — 540°± + 随机方向才有"随机翻飞"感
- ❌ 不要用 `setInterval` 清理过期红包（line 318 cleanup 没用，已用 CSS animation 控制消失）
- ❌ 不要忘了 `will-change` + `backface-visibility: hidden` — 没 GPU 加速会掉帧

## 红包雨顶部定位闪现修复（v2）

### 用户反馈
"对比下，顶部还有红包定位闪现问题"

### 根因（深入）
v1 修复了"scale 缩放造成的顶部卡顿"，但**仍存在"定位闪现"**：

```css
0%   { top: -50px; opacity: 0; }
8%   { /* top 自动插值 = -50px + 7.6% × 容器高 = -30px */ opacity: 1; }
100% { top: 100%; opacity: 0; }
```

8% 关键帧**没有显式声明 top**，浏览器自动插值：
- 0% 红包 top: -50px（在容器外）
- 8% 红包 top ≈ -30px（**仍然在容器外**！容器边缘到 -50px 算外部）
- 8% opacity: 1 → **红包突然在容器外完全可见，但 top 是 -30px，部分被裁剪**

**视觉效果**：红包"突然出现在容器顶部边缘"——用户感知是"定位闪现"。

### 修复方案：让 opacity 渐入时红包已完全在容器内

#### 关键改进
- **8% 必须显式声明 top: 0px**（红包已到达容器顶，但 opacity 仍 0）
- **新增 18% 关键帧**：top: 5%, opacity: 0.5（红包已在容器内 5%，开始半透明淡入）
- **25% 关键帧**：top: 20%, opacity: 1（完全可见 20% 位置）

#### 完整 keyframes（v2）
```css
@keyframes fallRedpacketNatural {
  0%   { top: -50px; opacity: 0; transform: rotate(0deg); }
  8%   { top: 0px; opacity: 0; transform: rotate(0.1×rot-step); }
  18%  { top: 5%; opacity: 0.5; transform: rotate(0.2×rot-step); }
  25%  { top: 20%; opacity: 1; transform: rotate(0.5×rot-step); }
  50%  { top: 55%; transform: rotate(1.0×rot-step); }
  75%  { top: 80%; transform: rotate(1.5×rot-step); }
  95%  { top: 100%; opacity: 1; transform: rotate(1.9×rot-step); }
  100% { top: 100%; opacity: 0; transform: rotate(2.0×rot-step); }
}
```

#### 三段式淡入淡出
| 阶段 | 进度 | top | opacity | 状态 |
|------|------|-----|---------|------|
| 容器外飘入 | 0% → 8% | -50px → 0px | 0 → 0 | 不可见，进入容器顶 |
| 半透明淡入 | 8% → 18% | 0px → 5% | 0 → 0.5 | 刚进入容器，开始淡入 |
| 完全可见 | 18% → 25% | 5% → 20% | 0.5 → 1 | 淡入完成 |
| 正常飘落 | 25% → 95% | 20% → 100% | 1 | 完全可见正常飘落 |
| 出容器淡出 | 95% → 100% | 100% → 100% | 1 → 0 | 离开容器边缘淡出 |

#### 关键洞察
**opacity 渐入起点必须比 top 进入容器晚一个阶段** — 这样：
- 红包先在容器外飘一段（不可见）
- 到达容器顶时**仍不可见**
- 真正进入容器内（top >= 0）后才开始淡入
- 用户看到的是"红包从容器内顶部淡入"，而不是"从容器外突然出现"

### 设计禁忌
- ❌ 不要在 0%/8% 关键帧**只声明 opacity 不声明 top** — 浏览器会自动插值，top 会停留在容器外
- ❌ 不要让 opacity 渐入期间 top < 0 — 用户会看到"容器外突然出现"
- ❌ 不要让 opacity 渐出期间 top = 100% — 红包会"突然消失"在底部边缘
- ❌ 不要省略 18% 中间关键帧 — 8%→25% 跨度太大，红包从 0px 跳到 20% 不够细腻

## 红包雨"不同位置闪烁"修复（v3）

### 用户反馈
"顶部红包不同位置闪烁是不是修不了"

### 根因（v2 修复后的遗留问题）

v2 修复了"8% 时红包在容器外（-30px）突然 opacity 1"的问题，但**仍存在"不同位置闪烁"**：

```css
0%   { top: -50px; opacity: 0; }
8%   { top: 0px;   opacity: 0; }      /* 到达容器顶，仍不可见 */
18%  { top: 5%;    opacity: 0.5; }    /* 容器内 5%，半透明 */
25%  { top: 20%;   opacity: 1; }      /* 完全可见 */
```

18% 关键帧 opacity 0.5 时红包在 top 5%（容器内），25% 时 opacity 1 时红包在 top 20%（容器内）。

**问题**：opacity 从 0.5 → 1 期间（18% → 25%，约 385ms），红包**在容器内顶部 5% → 20% 范围移动**。由于每个红包的 startX 不同（0-85%），**多个红包在容器内顶部不同 top 位置同时渐入**——视觉上是"不同 left 位置 + 不同 top 位置的红包同时渐入"。

**用户感知**：红包在容器内顶部不同位置"突然半透明出现再变实"——这就是"不同位置闪烁"。

### 修复方案：把 opacity 渐入/渐出完全移到容器外（v3）

**核心洞察**：
- 容器有 `overflow: hidden`，容器外的内容**天然不可见**
- 利用这个特性，让所有 opacity 渐变**都发生在容器外**（top < 0 或 top > 100%）
- 容器内的飘落阶段**始终保持 opacity 1**

#### v3 完整 keyframes
```css
@keyframes fallRedpacketNatural {
  0%   { top: -50px;  opacity: 0;   transform: translate3d(0,0,0) scale(1) rotate(0deg); }
  3%   { top: -30px;  opacity: 1;   transform: translate3d(0,0,0) scale(1) rotate(0deg); }
  25%  { top: 22%;    opacity: 1;   transform: translate3d(-10px,0,0) scale(1) rotate(calc(var(--rot-step,72deg)*.5)); }
  50%  { top: 55%;    opacity: 1;   transform: translate3d(0,0,0) scale(1) rotate(calc(var(--rot-step,72deg)*1)); }
  75%  { top: 80%;    opacity: 1;   transform: translate3d(10px,0,0) scale(1) rotate(calc(var(--rot-step,72deg)*1.5)); }
  98%  { top: 110%;   opacity: 1;   transform: translate3d(0,0,0) scale(1) rotate(calc(var(--rot-step,72deg)*1.95)); }
  100% { top: 110%;   opacity: 0;   transform: translate3d(0,0,0) scale(1) rotate(calc(var(--rot-step,72deg)*2)); }
}
```

#### 渐入渐出完全在容器外
| 阶段 | 进度 | top | opacity | 用户感知 |
|------|------|-----|---------|---------|
| 容器外淡入 | 0% → 3% | -50px → -30px | 0 → 1 | 不可见（容器外） |
| 容器内飘落 | 3% → 98% | -30px → 110% | 1 | **完全可见，无渐变** |
| 容器外淡出 | 98% → 100% | 110% → 110% | 1 → 0 | 不可见（容器外） |

#### 时间分配
- 0% → 3% = 165ms（5.5s duration 下）：红包在容器外完成 opacity 渐入
- 3% → 98% ≈ 5.2s：红包在容器内全程 opacity 1（**完全没有透明度变化**）
- 98% → 100% = 110ms：红包在容器外完成 opacity 渐出

#### 为什么彻底消除"不同位置闪烁"
- 之前（v2）：opacity 渐入期间红包**在容器内 0% → 20% 位置移动**——用户看到"容器内不同位置出现"
- 现在（v3）：opacity 渐入期间红包**完全在容器外**——用户根本看不到渐入过程
- 进入容器时（top = 0%）红包**已经是 opacity 1**——所以**所有红包进入容器时都是"突然完全可见"**，但因为这是**连续的飘落运动**（不是"突变"），用户感知是"红包从顶边进入"

#### 关键技巧
- **利用 overflow: hidden 天然遮罩**：容器外不可见不需要 opacity 0
- **容器内永远 opacity 1**：避免任何"渐入"造成的"位置闪烁"
- **渐入发生在 0% → 3% 极短时间窗口**（165ms）：用户感知不到

### 设计禁忌
- ❌ 不要让 opacity 渐入期间 top >= 0%（红包会在容器内不同位置"渐入"）
- ❌ 不要让 opacity 渐入/渐出阶段超过 5%（用户能感知到"突然出现/消失"）
- ❌ 不要在 keyframes 中间加 opacity 0.5（半透明状态，红包在容器内闪烁）
- ❌ 不要忽略 0%/3%/98%/100% 这四个关键点（它们是"容器外"的 4 个边界）
- ❌ 不要让容器无 overflow: hidden（红包会从顶飘到容器外继续飘，看起来很怪）
